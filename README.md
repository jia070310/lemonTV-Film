# lemonTV-Film

一款专为 **Android TV / 电视盒子** 设计的 **MacCMS 影视客户端**，支持首页分类浏览、筛选搜索、详情选集、记忆续播、跳过片头片尾、在线更新等功能。

- 包名：`com.lemon.yingshi.tv`
- 当前版本：**v1.0.5**（versionCode **10005**）
- 仓库地址：https://github.com/jia070310/lemonTV-Film
- 安装包：[Releases](https://github.com/jia070310/lemonTV-Film/releases)
- 成品APP服务器地址为空，需要用户自行部署MACCMS服务器，可以利用NAS部署网站，然后填入局域网地址测试通过就可以看了。也可以网上查找MACCMS服务器部署的网站，前提条件是这个服务器开启了API，要不然是提取不到数据的。
- 这里推荐一个网址 https://xsd.sdzyapi.com
- 本应用**不内置影视服务器**，需自行部署 MacCMS 或接入已开启 API 的站点（部署与开启方法见下文 [MacCMS 部署与 API 配置](#maccms-部署与-api-配置)）
- 也可使用他人已部署好的 MacCMS 站点，**前提是已开启视频 API**，否则无法拉取数据

## 功能特性

### 内容浏览
- **MacCMS 接入**：在设置中配置站点 API 地址，即可拉取影视数据
- **首页分区**：电视剧、电影、综艺、演唱会等分类展示
- **卡片骨架加载**：首页/筛选页先显示卡片框架与片名，封面逐张淡入
- **详情页秒开**：复用列表缓存数据，进入详情立即展示海报与简介
- **最新推荐**：首页展示 MacCMS 推荐等级（level=9）内容，支持进入推荐列表页浏览更多
- **筛选与搜索**：按类型、地区、年份等条件筛选；支持搜索历史
- **详情页**：展示简介、海报、选集与播放源；支持多集连续浏览
- **我的收藏**：首页右上角进入收藏页；详情页一键收藏/取消，按当前 MacCMS 服务器分别存储

### 播放体验
- **ExoPlayer 播放**：基于 Media3，支持常见点播格式
- **多线程 HLS 预缓存**：m3u8 分片 6 线程并行下载至本地缓存，减少卡顿
- **OkHttp + 磁盘缓存**：播放网络栈连接池复用，512MB 本地缓存加速二次播放
- **缓冲进度可视化**：进度条显示已缓冲/预缓存范围，预缓存时显示 `x/y` 分片计数
- **记忆续播**：可在设置中开关；再次打开从上次进度继续
- **变速播放**：播放器内切换播放倍速
- **选集切换**：播放器内分页选集；支持上一集 / 下一集
- **跳过片头 / 片尾**：按**整部剧集**统一配置，对所有集数生效
- **续播提示**：有历史进度时可选择「继续播放」或「从头开始」

### TV 端优化
- **Jetpack Compose for TV**：针对遥控器焦点与大屏布局优化
- **焦点导航**：方向键移动焦点，OK 键确认，返回键逐级退出
- **分阶段加载**：首页与筛选页优先展示可见内容，后台补全封面与列表，打开更快
- **控制栏自动隐藏**：播放中可呼出进度条与功能按钮
- **快速跳转到首页下方状态栏**：点击遥控器菜单键可以跳转到状态栏

### 其他
- **最近观看**：记录观看历史与本地封面截图
- **版本更新**：从 GitHub Releases 检测并下载新版本；首页顶部提示条支持遥控器聚焦，确认后直接下载，10 秒后自动消失
- **远程通知**：首页通知栏支持远程 JSON 配置

## 系统要求

- Android 5.0（API 21）及以上
- Android TV 或带遥控器的电视盒子
- 可访问所配置的 MacCMS 站点网络

## MacCMS 部署与 API 配置

lemonTV-Film 通过 MacCMS 的 **视频采集 API**（`api.php/provide/vod`）拉取分类、列表、详情与播放地址。部署好 MacCMS 并**开启视频 API** 后，在 APP 设置里填入站点根地址即可使用。

### 一、部署 MacCMS（苹果 CMS V10）

#### 环境要求

| 组件 | 建议版本 |
|------|----------|
| PHP | 7.4 ~ 8.1（需扩展：mysqli、curl、gd、mbstring、xml） |
| MySQL | 5.7 / 8.0 |
| Web 服务 | Nginx 或 Apache |

#### 方式 1：NAS / 局域网服务器（推荐家庭使用）

适合在群晖、威联通、飞牛 NAS 或家里一台常开电脑上建站，电视与手机在同一局域网即可访问。

1. **安装 Web 环境**
   - 群晖：套件中心安装 **Web Station**、**MariaDB/MySQL**、**PHP 7.4+**
   - 威联通：通过 **Container Station** 运行 MacCMS Docker 镜像，或安装 Web 服务器套件
   - Windows：可用 **phpStudy**、**宝塔面板** 等一键环境

2. **部署 MacCMS 源码**
   - 从 [MacCMS 官方](https://github.com/magicblack) 或可信渠道下载 **苹果 CMS V10** 源码
   - 解压到网站根目录（如 `/www/wwwroot/maccms` 或 NAS 的 `web/maccms`）
   - 浏览器访问 `http://NAS的IP:端口/install.php` 完成安装向导（数据库名、账号密码等）

3. **安装完成后**
   - 前台：`http://192.168.x.x:端口/`
   - 后台：`http://192.168.x.x:端口/admin.php`
   - **务必修改**默认管理员密码

4. **导入影视数据（可选）**
   - 后台 **采集** → 添加 **联盟资源** / **自定义接口**，绑定资源站并执行采集
   - 也可在后台 **视频** → **添加** 手动录入测试影片

#### 方式 2：云服务器 / 宝塔面板

1. 购买云服务器，安装 **宝塔 Linux 面板**
2. 面板中安装 **Nginx + MySQL + PHP 7.4**
3. 新建网站，将 MacCMS V10 源码上传到网站目录
4. 绑定域名（可选 HTTPS），访问 `http://你的域名/install.php` 安装
5. 在路由器或安全组放行 **80 / 443** 端口；电视端填写公网域名或内网穿透地址

#### 方式 3：Docker 一键部署

```bash
# 示例：使用社区 MacCMS Docker 镜像（镜像名以实际为准）
docker run -d \
  --name maccms \
  -p 7963:80 \
  -e MYSQL_HOST=db \
  -e MYSQL_DATABASE=maccms \
  maccms/maccms10:latest
```

安装后访问 `http://服务器IP:7963/install.php`，按提示完成初始化。

### 二、开启 API（必做）

APP **必须**能访问视频采集接口，否则首页无分类、无内容。按下面步骤在 MacCMS 后台开启。

#### 1. 开启「视频 API」（必须）

1. 登录 MacCMS 后台：`http://你的站点/admin.php`
2. 进入 **系统** → **开放API配置**（部分版本为 **系统** → **采集参数配置** → **视频API**）
3. 确认以下设置：

| 配置项 | 建议值 | 说明 |
|--------|--------|------|
| **视频API** | **开启** | 关闭后 APP 无法拉取任何数据 |
| **指定播放组** | 留空 | 留空表示输出全部播放组；若只填某一组，可能导致部分影片无播放地址 |
| **图片域名** | 留空或填站点域名 | 留空时使用相对路径；若封面不显示，可填 `http://你的站点域名` |
| **接口授权密码** | 留空 | 本 APP 当前按无密码的开放接口访问；若后台强制鉴权，需与 APP 配置一致 |

4. 保存后，在电脑浏览器测试（将 `192.168.0.16:7963` 换成你的地址）：

```
http://192.168.0.16:7963/api.php/provide/vod/?ac=list&pg=1&pagesize=5
```

正常应返回 JSON/XML，且 `code` 为 `1`，`list` 中有视频条目。

再测详情接口：

```
http://192.168.0.16:7963/api.php/provide/vod/?ac=detail&ids=1
```

（将 `1` 换成你站点里真实存在的 `vod_id`）

#### 2. 开启「公共 API」（可选，建议开启）

开启后 APP 可优先使用 REST 分类接口，分类树更准确；**不开启也能用**，APP 会自动回退到视频 API 的 `class` 字段解析分类。

1. 后台 **系统** → **开放API配置** → **公共API**
2. 将 **公共API** 设为 **开启**
3. 浏览器验证：

```
http://192.168.0.16:7963/api.php/type/get_list/
```

返回 `code: 1` 且含 `type_id`、`type_name` 等字段即成功。若返回 `closed` 或未开启，不影响基本使用，仅分类来源会走兼容模式。

#### 3. 首页「最新推荐」配置（可选）

APP 首页 **最新推荐** 读取 MacCMS **推荐等级 level=9** 的影片，与模板标签 `{maccms:vod level="9"}` 一致。

1. 后台 **视频** → 编辑需要推荐的影片
2. 将 **推荐** 设为 **9**（或后台「推荐等级」对应数值 9）
3. 保存后可在浏览器核对模板页是否有条目：

```
http://192.168.0.16:7963/index.php/vod/show/level/9.html
```

页面中能看到影片链接即表示推荐数据生效。

> **说明：** 部分站点的 `ac=list&level=9` 接口不会正确过滤等级，APP 已内置从上述模板页解析 ID 再批量拉取详情的兼容方案，无需额外配置。

### 三、在 lemonTV-Film 中连接 MacCMS

1. 电视/盒子安装 APP，进入 **设置** → **资源管理**
2. 在 **服务器地址** 填写 MacCMS **站点根地址**（规则如下）

| 正确示例 | 错误示例 |
|----------|----------|
| `http://192.168.0.16:7963` | `http://192.168.0.16:7963/api.php` |
| `https://maccms.example.com` | `http://192.168.0.16:7963/admin.php` |

3. 点击 **保存配置** → **测试连通性**
4. 状态显示 **已连接** 且出现 **分类数量** 后，返回首页即可浏览

局域网部署时，电视与 MacCMS 服务器需在同一网段，或已做端口转发/内网穿透。

### 四、API 自检清单

部署完成后，建议逐项在浏览器访问确认：

| 检测项 | 地址 | 期望结果 |
|--------|------|----------|
| 视频列表 | `/api.php/provide/vod/?ac=list&pg=1&pagesize=5` | `code=1`，有 `list` |
| 视频详情 | `/api.php/provide/vod/?ac=detail&ids=影片ID` | `code=1`，含封面 `vod_pic`、播放组 |
| 分类（可选） | `/api.php/type/get_list/` | `code=1`，有分类树 |
| 推荐（可选） | `/index.php/vod/show/level/9.html` | 页面含推荐影片链接 |
| 前台可访问 | `/` | 网站首页正常打开 |

### 五、常见问题（部署与 API）

| 现象 | 原因与处理 |
|------|------------|
| APP 提示「请先在设置中配置 MacCMS 服务器」 | 未保存服务器地址，或地址为空 |
| 测试连通性失败 | 检查 IP/端口、防火墙、PHP 是否运行；确认视频 API 已开启 |
| 已连接但首页无分类 | 站点尚无视频数据，先采集或手动添加影片 |
| 有列表但无封面 | 列表接口不含封面，属正常；APP 会后台补全。若始终无图，检查 **图片域名** 配置 |
| 有详情但无法播放 | 检查该片 **播放组** 是否有效、播放地址是否可访问 |
| 最新推荐为空 | 后台未有 **推荐等级 9** 的影片，或模板页无数据 |
| 电视无法访问 NAS 地址 | 确认电视与 NAS 同网段；用电视浏览器先访问 `http://NAS_IP:端口` 测试 |

## 使用教程

### 一、首次使用

1. **安装应用**  
   从 [Releases](https://github.com/jia070310/lemonTV-Film/releases) 下载 `app-release.apk`，用 U 盘或局域网工具安装到电视 / 盒子。

2. **配置 MacCMS 站点**  
   - 若还没有 MacCMS，先按 [MacCMS 部署与 API 配置](#maccms-部署与-api-配置) 完成部署并**开启视频 API**  
   - 首页底部导航进入 **设置** → 左侧选择 **资源管理**  
   - 在 **服务器地址** 输入 MacCMS 站点根地址，例如 `http://192.168.0.16:7963`  
   - 点击 **保存配置**，再点 **测试连通性** 确认连接成功（状态显示「已连接」并展示分类数量）  
   - 保存后返回首页，等待分类与影视数据加载

3. **自定义首页分类（可选）**  
   - **设置** → **首页设置** → **首页分类设置**  
   - 可开关各分类是否在首页显示、调整排序、配置二级类目  
   - 完成后点 **保存**

> 若首页提示「请先在设置中配置 MacCMS 服务器」，说明尚未保存有效站点地址。

---

### 二、首页浏览

首页底部有三个入口：

| 入口 | 说明 |
|------|------|
| **首页** | 浏览各分类影视内容 |
| **筛选** | 进入全站筛选页 |
| **设置** | 资源、播放、关于等配置 |

**首页内容区：**

- **最近播放**：有观看记录时自动显示，点击可续播；右侧 **更多** 进入完整列表
- **最新推荐**：展示站点推荐影片（11 张卡片 + 更多），点击进入推荐列表页
- **分类栏目**：按电视剧、电影、综艺等分区展示，每行最多 10 条
- **更多**：进入该分类的筛选页，浏览完整列表
- **影片卡片**：OK 键进入详情页
- **搜索**：顶部搜索图标，进入关键词搜索

**遥控器操作：**

- 方向键：移动焦点  
- OK 键：确认 / 进入  
- 返回键：退出当前层级  

---

### 三、筛选与搜索

#### 筛选页

从首页底部 **筛选** 或某分类的 **更多** 进入。

- **左侧**：一级分类（电视剧、电影、综艺等），打开后默认焦点在左上角
- **顶部**：类型、地区、年份、排序等筛选条件
- **右侧**：影片网格列表，OK 键进入详情
- **焦点**：侧栏 → 筛选条 → 卡片区逐步平移；卡片区支持左右切换
- 切换条件后自动刷新结果

#### 搜索页

- 输入片名关键词搜索 MacCMS 站点内容
- 自动记录 **搜索历史**，可一键 **清除历史**
- 点击结果进入详情页

---

### 四、详情页与播放

进入影片详情后可查看：

- 海报、片名、评分、简介
- **选集列表**（电视剧 / 综艺 / 动漫 / 纪录片等多集内容）
- **立即播放** 按钮

**播放逻辑：**

- 首次播放：从第一集或所选集开始
- 有历史进度且已开启「记忆续播」：弹出 **继续播放** / **从头开始** 选择
- 单集电影：直接播放

### 五、播放器

呼出控制栏后，可使用以下功能：

| 功能 | 说明 |
|------|------|
| **上一集 / 下一集** | 多集内容间切换 |
| **变速播放** | 0.25X ~ 2.0X 共 8 档倍速 |
| **选集** | 分页展示集数（每页 20 集），自动定位当前集 |
| **缓冲条** | 浅灰为已缓冲/预缓存范围，黄色为已播放进度 |
| **跳过** | 配置片头 / 片尾跳过（见下文） |

**跳过片头 / 片尾：**

- 在播放器中点 **跳过** 打开设置弹窗
- **生效范围：整部剧集**，对该剧所有集数统一生效
- 可分别设置 **片头时长**、**片尾时长**（秒）
- 可开关「自动跳过片头」「自动跳过片尾」
- 播放过程中可用 **将当前设为片头** / **将当前设为片尾** 快速标记时间点
- 设置后点 **保存设置**；换集后自动应用已保存配置

**播放结束：** 多集内容播放完成后自动跳转下一集。

**播放器快捷键：**

| 按键 | 功能 |
|------|------|
| OK / 中心键 | 播放 / 暂停；显示控制栏 |
| 左 / 右 | 快退 / 快进（时长见设置，默认 15 秒） |
| 上 / 下 | 控制栏内移动焦点 |
| 返回 | 隐藏控制栏 / 关闭弹窗 / 退出播放 |

### 六、设置中心

设置页左侧分为四个模块：

#### 1. 资源管理

- **MacCMS 服务器**：配置站点地址、保存、测试连通性
- 连接成功后显示站点名称、分类数量等信息

#### 2. 首页设置

- **首页分类设置**：控制首页显示哪些分类、排序及二级类目
- **清除缓存数据**：重置首页分类显示与排序（不影响观看记录）

#### 3. 播放设置

- **快进快退时长**：遥控器左右键跳转秒数（15 / 20 / 25 / 30 / 35 秒）
- **记忆续播功能**：开关自动记录与恢复播放进度
- **清空最近播放记录**：删除全部观看历史（不可恢复）

#### 4. 关于应用

- **版本更新**：检测 GitHub Releases 新版本并下载安装
- **应用统计**：查看累计播放时长
- 设置图标上的红点表示有新版本可用

### 七、常见问题

| 现象 | 处理建议 |
|------|----------|
| 首页无内容 | 检查 MacCMS 地址是否正确；确认已 [开启视频 API](#maccms-部署与-api-配置)；先 **测试连通性** 再保存 |
| 测试连通性失败 | 用电脑浏览器访问 `/api.php/provide/vod/?ac=list&pg=1&pagesize=1` 排查；检查防火墙与网络 |
| 搜索 / 筛选无结果 | 确认网络可达 MacCMS 站点；站点是否有视频数据；尝试换关键词或放宽筛选条件 |
| 播放失败 | 检查该片播放源是否有效；后台播放组是否配置正确；稍后重试或换集 |
| 跳过片头片尾不生效 | 确认已在 **跳过** 弹窗中开启对应开关并 **保存设置** |
| 无法检测更新 | 确认电视可访问 GitHub；手动从 Releases 页下载安装 |

## 项目结构

```
app/src/main/java/com/lemon/yingshi/tv/
├── data/           # MacCMS API、本地数据库、偏好设置
```
