# lemonTV-Film

一款专为 **Android TV / 电视盒子** 设计的 **MacCMS 影视客户端**，支持首页分类浏览、筛选搜索、详情选集、记忆续播、跳过片头片尾、在线更新等功能。

- 包名：`com.lemon.yingshi.tv`
- 当前版本：**v1.0.0**（versionCode **10000**）
- 仓库地址：https://github.com/jia070310/lemonTV-Film
- 安装包：[Releases](https://github.com/jia070310/lemonTV-Film/releases)
成品APP服务器地址为空，需要用户自行部署MACCMS服务器，可以利用NAS部署网站，然后填入局域网地址测试通过就可以看了。
这里推荐一个网址 https://xsd.sdzyapi.com
## 功能特性

### 内容浏览
- **MacCMS 接入**：在设置中配置站点 API 地址，即可拉取影视数据
- **首页分区**：电视剧、电影、综艺、演唱会等分类展示
- **筛选与搜索**：按类型、地区、年份等条件筛选；支持搜索历史
- **详情页**：展示简介、海报、选集与播放源；支持多集连续浏览

### 播放体验
- **ExoPlayer 播放**：基于 Media3，支持常见点播格式
- **记忆续播**：可在设置中开关；再次打开从上次进度继续
- **变速播放**：播放器内切换播放倍速
- **选集切换**：播放器内分页选集；支持上一集 / 下一集
- **跳过片头 / 片尾**：按**整部剧集**统一配置，对所有集数生效
- **续播提示**：有历史进度时可选择「继续播放」或「从头开始」

### TV 端优化
- **Jetpack Compose for TV**：针对遥控器焦点与大屏布局优化
- **焦点导航**：方向键移动焦点，OK 键确认，返回键逐级退出
- **控制栏自动隐藏**：播放中可呼出进度条与功能按钮
- **快速跳转到首页下方状态栏**：点击遥控器菜单键可以跳转到状态栏

### 其他
- **最近观看**：记录观看历史与本地封面截图
- **版本更新**：从 GitHub Releases 检测并下载新版本
- **远程通知**：首页通知栏支持远程 JSON 配置

## 系统要求

- Android 5.0（API 21）及以上
- Android TV 或带遥控器的电视盒子
- 可访问所配置的 MacCMS 站点网络

## 快速开始

### 安装

从 [Releases](https://github.com/jia070310/lemonTV-Film/releases) 下载最新 `app-release.apk` 安装到电视或盒子。

### 配置 MacCMS

1. 打开应用，进入 **设置**
2. 在 **MacCMS 配置** 中填写站点 API 地址与密钥
3. 保存后返回首页，等待分类与内容加载

## 使用教程

### 一、首次使用

1. **安装应用**  
   从 [Releases](https://github.com/jia070310/lemonTV-Film/releases) 下载 `app-release.apk`，用 U 盘或局域网工具安装到电视 / 盒子。

2. **配置 MacCMS 站点**  
   - 首页底部导航进入 **设置** → 左侧选择 **资源管理**  
   - 在 **服务器地址** 输入 MacCMS 站点根地址，例如 `https://your-maccms.com`  
   - 点击 **保存配置**，再点 **测试连通性** 确认连接成功（状态显示「已连接」并展示分类数量）  
   - 保存后返回首页，等待分类与影视数据加载

3. **自定义首页分类（可选）**  
   - **设置** → **首页设置** → **首页分类设置**  
   - 可开关各分类是否在首页显示、调整排序、配置二级类目  
   - 完成后点 **保存**

> 若首页提示「请先在设置中配置 MacCMS 服务器」，说明尚未保存有效站点地址。

---

### 二、首页浏览

首页底部有三个入口：

| 入口 | 说明 |
|------|------|
| **首页** | 浏览各分类影视内容 |
| **筛选** | 进入全站筛选页 |
| **设置** | 资源、播放、关于等配置 |

**首页内容区：**

- **最近播放**：有观看记录时自动显示，点击可续播；右侧 **更多** 进入完整列表
- **分类栏目**：按电视剧、电影、综艺等分区展示，每行最多 10 条
- **更多**：进入该分类的筛选页，浏览完整列表
- **影片卡片**：OK 键进入详情页
- **搜索**：顶部搜索图标，进入关键词搜索

**遥控器操作：**

- 方向键：移动焦点  
- OK 键：确认 / 进入  
- 返回键：退出当前层级  

---

### 三、筛选与搜索

#### 筛选页

从首页底部 **筛选** 或某分类的 **更多** 进入。

- **左侧**：一级分类（电视剧、电影、综艺等）
- **顶部**：类型、地区、年份、排序等筛选条件
- **右侧**：影片网格列表，OK 键进入详情
- 切换条件后自动刷新结果

#### 搜索页

- 输入片名关键词搜索 MacCMS 站点内容
- 自动记录 **搜索历史**，可一键 **清除历史**
- 点击结果进入详情页

---

### 四、详情页与播放

进入影片详情后可查看：

- 海报、片名、评分、简介
- **选集列表**（电视剧 / 综艺 / 动漫 / 纪录片等多集内容）
- **立即播放** 按钮

**播放逻辑：**

- 首次播放：从第一集或所选集开始
- 有历史进度且已开启「记忆续播」：弹出 **继续播放** / **从头开始** 选择
- 单集电影：直接播放

---

### 五、播放器

呼出控制栏后，可使用以下功能：

| 功能 | 说明 |
|------|------|
| **上一集 / 下一集** | 多集内容间切换 |
| **变速播放** | 0.25X ~ 2.0X 共 8 档倍速 |
| **选集** | 分页展示集数（每页 20 集），自动定位当前集 |
| **跳过** | 配置片头 / 片尾跳过（见下文） |

**跳过片头 / 片尾：**

- 在播放器中点 **跳过** 打开设置弹窗
- **生效范围：整部剧集**，对该剧所有集数统一生效
- 可分别设置 **片头时长**、**片尾时长**（秒）
- 可开关「自动跳过片头」「自动跳过片尾」
- 播放过程中可用 **将当前设为片头** / **将当前设为片尾** 快速标记时间点
- 设置后点 **保存设置**；换集后自动应用已保存配置

**播放结束：** 多集内容播放完成后自动跳转下一集。

**播放器快捷键：**

| 按键 | 功能 |
|------|------|
| OK / 中心键 | 播放 / 暂停；显示控制栏 |
| 左 / 右 | 快退 / 快进（时长见设置，默认 15 秒） |
| 上 / 下 | 控制栏内移动焦点 |
| 返回 | 隐藏控制栏 / 关闭弹窗 / 退出播放 |

---

### 六、设置中心

设置页左侧分为四个模块：

#### 1. 资源管理

- **MacCMS 服务器**：配置站点地址、保存、测试连通性
- 连接成功后显示站点名称、分类数量等信息

#### 2. 首页设置

- **首页分类设置**：控制首页显示哪些分类、排序及二级类目
- **清除缓存数据**：重置首页分类显示与排序（不影响观看记录）

#### 3. 播放设置

- **快进快退时长**：遥控器左右键跳转秒数（15 / 20 / 25 / 30 / 35 秒）
- **记忆续播功能**：开关自动记录与恢复播放进度
- **清空最近播放记录**：删除全部观看历史（不可恢复）

#### 4. 关于应用

- **版本更新**：检测 GitHub Releases 新版本并下载安装
- **应用统计**：查看累计播放时长
- 设置图标上的红点表示有新版本可用

---

### 七、常见问题

| 现象 | 处理建议 |
|------|----------|
| 首页无内容 | 检查 MacCMS 地址是否正确，先 **测试连通性** 再保存 |
| 搜索 / 筛选无结果 | 确认网络可达 MacCMS 站点；尝试换关键词或放宽筛选条件 |
| 播放失败 | 检查该片播放源是否有效；稍后重试或换集 |
| 跳过片头片尾不生效 | 确认已在 **跳过** 弹窗中开启对应开关并 **保存设置** |
| 无法检测更新 | 确认电视可访问 GitHub；手动从 Releases 页下载安装 |

### 构建

```bash
git clone https://github.com/jia070310/lemonTV-Film.git
cd lemonTV-Film

# 调试包
./gradlew assembleDebug

# 发布包（需项目根目录 lomenTV.jks 签名文件）
./gradlew assembleRelease
```

Release 签名配置见 `app/build.gradle.kts` 中的 `signingConfigs.release`。

## 项目结构

```
app/src/main/java/com/lemon/yingshi/tv/
├── data/           # MacCMS API、本地数据库、偏好设置
├── domain/         # 业务模型与服务（播放、历史、更新等）
├── ui/             # Compose 界面（首页、详情、播放器、设置）
├── di/             # Hilt 依赖注入
└── service/        # 广播接收器等
```

## 技术栈

- Kotlin
- Jetpack Compose + Compose for TV
- Hilt
- Room
- Media3 (ExoPlayer)
- OkHttp + Kotlin Serialization
- Navigation Compose
